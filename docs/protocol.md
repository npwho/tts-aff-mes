# Extension <-> Native WebSocket Protocol

Server: hosted by the native Python tool at `ws://127.0.0.1:8765`.
Client: the extension's `background.js` service worker.
Encoding: JSON, one object per message, always with a `type` field.

Request/response pairs carry a `corrId` (string, generated by the requester) echoed back in the
response so the native side can match async replies without ambiguity.

The extension NEVER dispatches synthetic input events into the page. It only reads the DOM
(`querySelector`, `getBoundingClientRect`, `MutationObserver`, passive capture-phase listeners)
and reports what it observes. All real mouse/keyboard action happens on the native side via OS-level
input (pyautogui/pynput).

## Handshake

**`hello`** (ext -> native) — sent on WS open and again on every SPA navigation the content script detects.
```json
{"type": "hello", "extVersion": "0.1.0", "tabId": 123, "url": "https://.../messages"}
```

**`hello_ack`** (native -> ext)
```json
{"type": "hello_ack", "mode": "idle|calibrating|recording|replaying", "serverVersion": "0.1.0"}
```

## Health

**`ping`** (native -> ext, every ~15s) / **`pong`** (ext -> native, immediate reply).
If a `pong` doesn't arrive within 5s, native marks the connection stale, pauses any in-flight
run, and surfaces "disconnected" in the GUI rather than continuing to fire OS input blind.

**`captcha_detected`** (ext -> native, unsolicited, any time) — extension's MutationObserver
spotted a captcha container/iframe anywhere on the page.
```json
{"type": "captcha_detected", "detail": "iframe[src*=captcha] appeared"}
```
Native response: immediately pause the whole run (not just skip one username) and alert the user.

**`error`** (either direction) — anything unexpected.
```json
{"type": "error", "context": "replay_locate", "message": "selector threw"}
```

## Calibration phase

**`calibration_request_point`** (native -> ext)
```json
{"type": "calibration_request_point", "corrId": "cal-1", "pointId": 1, "viewportX": 50, "viewportY": 50}
```
Extension inserts a small fixed-position marker DOM element at that viewport coordinate
(pure rendering — not a synthetic input event) and replies:

**`calibration_marker_ready`** (ext -> native)
```json
{
  "type": "calibration_marker_ready",
  "corrId": "cal-1",
  "pointId": 1,
  "viewportX": 50,
  "viewportY": 50,
  "windowInnerWidth": 1512,
  "windowInnerHeight": 864,
  "devicePixelRatio": 1.5,
  "screenX": 40,
  "screenY": 30
}
```
The user then hovers their real mouse over the marker and presses F8 (or clicks "Capture" in the
Tk GUI); native reads the real cursor position via `pyautogui.position()`. Repeated for a second,
far-apart point to solve the 2-point affine transform (handles zoom + DPI scaling together).

**`calibration_complete`** (native -> ext) — informational, extension removes markers.
```json
{"type": "calibration_complete", "transform": {"scaleX": 1.5, "scaleY": 1.5, "offsetX": 40, "offsetY": 30}}
```

Native persists the transform to `native/storage/calibration.json` together with the
`screenX`/`screenY` observed at calibration time, used later for drift detection.

## Recording phase

**`start_recording`** (native -> ext)
```json
{"type": "start_recording"}
```

**`record_event`** (ext -> native) — emitted for each real, user-driven `click`/`focus`/`input`
event captured passively. Never synthesized.
```json
{
  "type": "record_event",
  "eventType": "click",
  "selector": {
    "strategy": "data-testid",
    "value": "[data-testid='new-message-btn']",
    "attributes": {"data-testid": "new-message-btn", "aria-label": "New message"},
    "textContent": "New message",
    "ancestorPath": "div.message-panel > button"
  },
  "rectViewport": {"x": 1200, "y": 40, "w": 120, "h": 32},
  "tag": null,
  "timestamp": 1730000000000
}
```
Native's recorder tags certain observed sequences specially when serializing to
`action_script.json`:
- `HOVER_THEN_CLICK` — a click on an element that only became visible/positioned after a preceding
  real mouseover on a sibling/ancestor row (the "Chat" button case).
- `PASTE_MULTILINE_THEN_ENTER` — a focus on the message box followed by a paste and the flow ending
  (message sent).

**`stop_recording`** (native -> ext) — user clicked "Stop Recording & Save" in the GUI once the
first creator's message was sent.
```json
{"type": "stop_recording"}
```

## Replay phase (per username)

**`replay_locate`** (native -> ext) — "does this selector resolve right now, and if so, what's its rect?"
```json
{"type": "replay_locate", "corrId": "r-42", "stepId": 3, "selectorDescriptor": {...}, "username": "creator123"}
```

**`replay_locate_result`** (ext -> native)
```json
{
  "type": "replay_locate_result",
  "corrId": "r-42",
  "stepId": 3,
  "found": true,
  "rectViewport": {"x": 300, "y": 220, "w": 260, "h": 40},
  "confidence": "exact|fallback",
  "matchedBy": "data-testid|id|text|structural"
}
```
or
```json
{"type": "replay_locate_result", "corrId": "r-42", "stepId": 3, "found": false, "reason": "no_dom_match"}
```

**`replay_locate_search_result`** (native -> ext) — specialized query after pasting a username
into the search box.
```json
{"type": "replay_locate_search_result", "corrId": "r-43", "username": "creator123"}
```

**`search_result_status`** (ext -> native)
```json
{"type": "search_result_status", "corrId": "r-43", "status": "found", "rectViewport": {...}}
```
```json
{"type": "search_result_status", "corrId": "r-43", "status": "not_found"}
```
```json
{"type": "search_result_status", "corrId": "r-43", "status": "ambiguous", "count": 2, "exactMatchRect": null}
```
On `not_found`: native logs `SKIPPED_NOT_FOUND`, runs `reset_state`, moves to the next username —
no pause, per the chosen failure-handling behavior.

**`get_text_content`** (native -> ext) — read-only readback, used to verify a paste landed before
pressing Enter.
```json
{"type": "get_text_content", "corrId": "r-44", "selectorDescriptor": {...}}
```

**`text_content_result`** (ext -> native)
```json
{"type": "text_content_result", "corrId": "r-44", "text": "Hi! ...", "length": 128}
```

**`send_verify`** (native -> ext) — after the real Enter keypress, confirm the message actually sent.
```json
{"type": "send_verify", "corrId": "r-45", "expectedTextPrefix": "Hi!"}
```

**`send_verify_result`** (ext -> native)
```json
{"type": "send_verify_result", "corrId": "r-45", "sent": true}
```
If `sent: false` after a timeout (default 4s), native logs `FAILED_SEND_UNCONFIRMED`.

**`reset_state`** (native -> ext, informational) — native performed a real Escape keypress +
click on a neutral area; asks the extension to confirm no dialog/modal remains open.
```json
{"type": "reset_state", "corrId": "r-46"}
```

**`verify_reset`** (ext -> native)
```json
{"type": "verify_reset", "corrId": "r-46", "dialogOpen": false}
```
If `dialogOpen: true` after native's retry budget (default 3 attempts), native pauses the entire
run — a stuck dialog risks the next username's clicks hitting the wrong target.

## Result status values written to the CSV log

`SENT`, `SKIPPED_NOT_FOUND`, `FAILED_SEND_UNCONFIRMED`, `FAILED_UI_STUCK`, `ABORTED`,
`DONE (recorded)` (the first, manually-run creator during Phase B).
